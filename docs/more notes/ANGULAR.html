State management

There is no NgRx. Client state is centralized in:

shared/ipac.session.info.ts (iPacSessionInfo)

  • Holds: Opportunity, IPUser, authority levels, premium rates, feature flags (MOH, delegate post, pricing adequacy)
  • Exposes BehaviorSubject streams: currentOpportunity, displaySpinner, getCostChangeConfig(), etc.
  • All HTTP via POST(url, data) ÃƒÂ¢Ã¢â‚¬Â Ã¢â‚¬â„¢ environment.BaseUrl + url

Feature services hold tab-specific dirty flags and cached data:

  • ClaimDataService ÃƒÂ¢Ã¢â€šÂ¬Ã¢â‚¬Â claim experience/projection
  • TargetLRDataService ÃƒÂ¢Ã¢â€šÂ¬Ã¢â‚¬Â target loss ratio changes
  • OpportunityService ÃƒÂ¢Ã¢â€šÂ¬Ã¢â‚¬Â aggregates child services for save/post/submit

HTTP pattern

// Typical call

Smart vs presentational components

The codebase predominantly uses smart components (inject services directly). Shared presentation is limited to SharedModule exports and common pipes.

Forms and validation

  • Template-driven forms (FormsModule)
  • Validation often in services: OpportunityService.validateClaimExprince(), etc.
  • User feedback via SweetAlert2 and ngx-toastr
