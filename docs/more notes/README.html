Bupa iPAC

Insurance Pricing and Administration for Corporate — enterprise pricing, renewal, claims analysis, and premium posting for Bupa Saudi Arabia corporate health insurance, integrated with Microsoft Dynamics CRM.

Overview

Item Description
Purpose Price corporate group health policies: load CRM opportunities, model claims and loss ratios, calculate premiums, enforce authority-based approvals, and post rates back to CRM
Users Corporate pricing/underwriting staff, iPAC administrators (factor configuration), reports administrators
Frontend Angular 18 SPA embedded in ASP.NET MVC Razor views
Backend .NET Framework 4.7.2, ASP.NET MVC 5
Data SQL Server via Dapper and stored procedures
CRM Dynamics CRM over WCF (net.tcp)

Request flow: UI component → Angular service → iPacSessionInfo.POST() → MVC controller → service layer → repository → stored procedure → database (and/or CRM).

Authentication: IIS Windows Authentication, Active Directory department check, CRM user/role mapping into ASP.NET session (SessionManager.IPUser). The Angular app does not use JWT; it relies on the server session cookie.

Prerequisites

  • Visual Studio 2019+ (or compatible) with ASP.NET and web development workload
  • .NET Framework 4.7.2
  • Node.js (LTS recommended) and npm — for the Angular client
  • SQL Server access to iPAC and technical-result databases
  • IIS / IIS Express with Windows Authentication enabled
  • Network access to CRM WCF endpoint, IBM API Gateway (URL decrypt), and other integrations configured in Web.config (environment-specific)

Getting started

1. Configure the backend

  1. Clone or open the solution.
  2. Edit Bupa.Ipac.Web\Web.config:
    • IpacConStr — main iPAC database
    • TRConStr — technical result staging database
    • CRM, IBM gateway, Elasticsearch, and AD settings under appSettings / system.serviceModel
  3. Do not commit real credentials; use local or team-specific config transforms where available.

2. Build and run the web application

  1. Set Bupa.Ipac.Web as the startup project.
  2. Restore NuGet packages (Visual Studio or nuget restore).
  3. Build the solution (F6).
  4. Run under IIS Express or IIS (typical dev URL: http://localhost:14314/).

Users must be on the corporate network with a valid Windows account in an authorized AD department and a mapped CRM role.

3. Build the Angular client

cd Bupa.Ipac.Web
npm install
Command Environment
npm run start Dev server (ng serve) — optional; production uses MVC host
npm run build-dev Development build → Scripts/ClientApp
npm run build-sit SIT
npm run build-uat UAT
npm run build-preprod Pre-production
npm run build-prod Production

Angular build output is written to Bupa.Ipac.Web/Scripts/ClientApp (see angular.json).

4. Entry point from CRM

Users typically open iPAC from CRM via an encrypted link:

  • MVC: Home/Index?id={encryptedOpportunityId}
  • Angular bootstraps via <app-root> in Views/Home/Index.cshtml with OpportunityUniqueNo, ReadOnly, and related attributes.

Deep links use base href /ngx (e.g. /ngx/opportunity, /ngx/projection).

Backend API surface

The SPA calls MVC controller actions with HTTP POST and JSON bodies, not a separate Web API layer:

POST {BaseUrl}{Controller}/{Action}

Examples:

Endpoint Purpose
Opportunity/GetAll Load opportunities and current user
Opportunity/GetIBMDecryptResponse Decrypt CRM deep-link parameter
Opportunity/SaveOpportunity Persist pricing state
Opportunity/CalaculatePremium Calculate premiums
Opportunity/PostPremium Post rates to CRM
Opportunity/SubmitForApproval Start approval workflow
Claim/GetClaimExperience, Claim/GetClaimProjection Claims data
TechnicalResultReport/* Technical result tabs (TR database)

Main controllers: OpportunityController, ClaimController, TechnicalResultReportController, SummaryReportController, UnderwritingController, NoteController, AuditController, and others under Bupa.Ipac.Web\Controllers/.

Admin area: Bupa.Ipac.Web/Areas/Admin/ — factor uploads, renewal configuration, authority matrices (Kendo UI grids). Route prefix: /Admin/{controller}/{action}.

Opportunity lifecycle

Statuses are defined in Bupa.iPAC.Entities/Custom/EnumType.cs (OpportunityStatus):

New → InProgress → SentToManager / PenddingApproval → Approve / Reject → Posted (and renewal-specific states such as RenewalNew, RenewalSaved).

Typical flow:

  1. Open opportunity from CRM
  2. Enter or adjust claims experience, projection, target loss ratio, cost-change factors
  3. Calculate and save premium
  4. Submit for approval if changes exceed user authority
  5. Post premium to CRM when approved

Windows Service

Bupa.Ipac.WindowsService runs as a Topshelf service (Bupa.Ipac.WindowsService):

  • Sync CRM opportunities and user roles into iPAC
  • Send factor-upload reminder emails

Deploy and configure separately from the web application.

Configuration reference

Setting Location Purpose
IpacConStr Web.config Main database
TRConStr Web.config Technical result staging database
appSettings Web.config CRM address, IBM gateway, ELK, roles
system.serviceModel Web.config WCF CRM client endpoint
environment.*.ts Bupa.Ipac.Web/src/environments/ Angular API base URL

Dependency injection: App_Start/AutofacBootstrap.cs registers services and both UnitOfWork instances.

Startup: Global.asax.cs — Autofac, routes, bundles, log4net.

Debugging tips

Issue What to check
Session timeout / 404 on API IIS session, SessionExpireFilter, cookie domain
Unauthorized AD department vs AuthorizedDepartments, CRM user disabled
Blank opportunity Opportunity/GetIBMDecryptResponse, IBM gateway config
Technical result empty TRConStr, TechnicalResultID on opportunity
Post to CRM fails WCF endpoint, CRM validation messages in logs

Useful breakpoints:

  • Frontend: app.component.ts (startup), opportunity.service.ts (saveOpportunity)
  • Backend: HomeController.GetAuthenticatedUser, OpportunityController.SaveOpportunity, OpportunityService.SaveOpportunity

Deployment notes

  • Host Bupa.Ipac.Web on IIS with Windows Authentication.
  • Build Angular with the target environment (npm run build-prod, etc.) before publishing.
  • Publish the MVC site including Scripts/ClientApp.
  • Install and start Bupa.Ipac.WindowsService on appropriate servers.
  • No Docker or Kubernetes manifests are included in this repository.
